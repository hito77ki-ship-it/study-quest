# Tier2 簿記3級バッチ3（決算整理・その他7本）ファクトチェック結果

**日付：** 2026年7月8日
**決定者：** CEO ジョブズ / 取締役承認
**ステータス：** 採用

## 背景

Tier2バッチ3として、決算整理・その他7本（経過勘定／経過勘定の応用／消費税／法人税等／株式会社／商品有高帳・売上原価／給料）を並行フォークでファクトチェックした。これで簿記3級のTier2（21本）が完了する。

## リサーチ要点

| 記事 | 結果 |
|------|------|
| boki3-keika.html（経過勘定） | **誤り**：前受収益の例題で月数カウントミス。決算日3月31日なのに、当期に含めるべき3月分を誤って翌期側にカウントしている（記事自身が「月数のカウントミスに注意」と警告している、まさにそのミスに陥っていた） |

| boki3-shohi.html（消費税） | 軽微：冒頭のpoint-cardが「税抜方式のみ出題される」と言いながら直後に税込方式の話をしており、字面上自己矛盾に見える。計算・仕訳自体は正確 |
| boki3-hojinzei.html（法人税等） | 問題なし。2026-07-08修正済みのboki3-sonota.html（事業税の扱い）とも整合 |

| boki3-shohinuriage.html（商品有高帳・売上原価） | 軽微〜中程度：移動平均法の例題で平均単価が割り切れない数値設定（206.666…→207に丸め）になっており、丸めた単価で数量×単価を計算すると表の金額欄と一致しない |

## 決定内容

1. `boki3-keika.html`：前受収益の例題を、決算日（3月31日）基準で正しい月数（当期5か月・翌期1か月）に修正
2. `boki3-keika2.html`：タイトル・meta descriptionを本文の実際の内容（毎期同額の経過勘定の応用パターン）に合わせて修正
3. `boki3-shohi.html`：point-cardの表現を、warn-box・FAQと一貫した言い回しに修正
4. `boki3-kabushiki.html`：利益準備金の積立上限ルールを「資本準備金＋利益準備金の合計が資本金の1/4」に修正
5. `boki3-shohinuriage.html`：移動平均法の例題の数値を、割り切れる組み合わせに変更するか、単価・金額の整合性を再計算

## ジョブズの所見

経過勘定の記事で「月数カウントミスに気をつけろ」と警告しながら自分がそのミスをしているのは、正直笑えないブラックジョークだが、これこそAIが一発で書いた記事にありがちな失敗——「注意点として言語化はできるが、それを自分の例題に適用し忘れる」というパターン。人間の熟練講師なら書きながら気づくミスだが、ここは仕組み（ファクトチェック）でカバーするしかない。

タイトルと本文の不一致（keika2）は、ゼロから簿記シリーズのときと全く同じ種類のバグ。1回で終わらせず、今後の記事作成・リライトでは「タイトルを最後に本文と突き合わせる」チェックを標準化すべき教訓が2回連続で出た。

商品有高帳の移動平均法は、割り切れない数値をそのまま使ったことが根本原因。実務的な数値設計のミスで、次に新しい例題を作るときは「割り算した後に検算する」を徹底する。

## 影響

- 3級の決算整理系（経過勘定・株式会社・商品有高帳）の正確性が向上し、Tier2 3級バッチ（21本）が完了する
- 残るTier2は2級20本（商業簿記9本・工業簿記11本）
