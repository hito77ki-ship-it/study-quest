# Tier2 簿記3級バッチ2（帳簿・伝票系7本）ファクトチェック結果

**日付：** 2026年7月8日
**決定者：** CEO ジョブズ / 取締役承認
**ステータス：** 採用

## 背景

Tier2ファクトチェックのバッチ2として、帳簿・伝票系7本（補助簿／伝票／仕訳日計表／試算表／精算表／銀行勘定調整表／三分法）を並行フォークでファクトチェックした。

## リサーチ要点

| 記事 | 結果 |
|------|------|
| boki3-hojosho.html（補助簿） | **誤り**：冒頭で「補助簿6種類」と説明しながら、後半では実質11種類（売上帳・仕入帳を含む）を列挙しており矛盾。取引×補助簿の対応表からも売上帳・仕入帳の列が丸ごと抜けている |
| boki3-denpyo.html（伝票） | **誤り**：「手形の割引」の例題が2019年度の出題区分改定で2級以上の範囲に移動済み。現在の3級には不要な内容 |
| boki3-nikkei.html（仕訳日計表） | **誤り**：補助元帳（得意先元帳・仕入先元帳）への転記元を「仕訳日計表」としているが、正しくは「伝票」から個別転記すべき。直前の警告ボックスの説明と自己矛盾している |
| boki3-shisanhyo.html（試算表） | 問題なし。数値の作り込みも精度が高い |
| boki3-seisanhyo.html（精算表） | **誤り（重大）**：メインの8桁精算表の例題で、試算表欄・B/S欄の貸方合計が個別科目の合計と一致しない（借方1,100・貸方1,550など）。複式簿記の大前提である「貸借一致」が例題自体で崩れている。コピペミスの可能性 |
| boki3-ginko.html（銀行勘定調整表） | **誤り2件**：①FAQで「未渡小切手は銀行側から減算」としているが、本文の分類・計算例と矛盾（正しくは自社帳簿側のみの修正）②meta description/FAQが「時間外入金」を4パターンの1つとして挙げているが、本文には存在せず実際の4つ目は「誤記入・未通知」 |
| boki3-sanpou.html（三分法） | 軽微な要検証：「分記法は試験範囲外」は言い過ぎ（実態は「ほとんど出題されない」）。出典不明な章番号（「第6章：固定資産の売却」）への言及あり |

Sources: [2019年度の日商簿記3級の出題範囲の改定について（パブロフ簿記）](https://pboki.com/re/2019reboki3.html)／[仕訳日計表とは（いぬぼき）](https://inuboki.com/3q-kouza/chapter8-2/)／[未取立小切手と未取付小切手の違い（パブロフ簿記）](https://pboki.com/nisho2/mi/mi.html)

## 決定内容

1. `boki3-hojosho.html`：「6種類」の表記を実態（売上帳・仕入帳を含む）に合わせて修正し、対応表に売上帳・仕入帳の列を追加
2. `boki3-denpyo.html`：手形の割引の例題を削除（3級の出題範囲外のため）
3. `boki3-nikkei.html`：補助元帳への転記元の説明を「伝票から個別転記」に修正
4. `boki3-seisanhyo.html`：メイン例題の試算表欄・B/S欄の貸方合計を、個別科目の合計と一致するよう修正
5. `boki3-ginko.html`：①FAQの未渡小切手の説明を本文と整合させる②meta description/FAQの「時間外入金」を本文が実際に解説している「誤記入・未通知」に統一
6. `boki3-sanpou.html`：「試験範囲外」を「ほとんど出題されない」に修正し、出典不明な章番号の言及を削除

## ジョブズの所見

7本中6本に問題があったのは、Tier1・バッチ1と比べて明らかに悪い数字。ただしこれは記事群全体の質が落ちているというより、「帳簿・伝票・精算表」という、複数の帳簿間の対応関係を説明する記事群特有の複雑さが原因だと見ている。対応表・転記の説明・貸借合計のような「表と表の整合性」は、AIが一発で書くと崩れやすい典型パターン。

精算表の計算ミスは今回で一番重い。これは正に、以前ユーザーの後輩（Yuki.K）が「精算表・決算整理後残高試算表が難しい」と言っていた、まさにその記事だ。貸借が合わない例題を見せられた受験生は、自分の計算が間違っているのか記事が間違っているのか分からなくなり、一番学習効果が高いはずの記事が一番信頼を損なうという最悪のパターンになる。優先して直す。

補助簿の「6種類なのに11種類」問題も根が深い。数え方の矛盾は、記事を書いた時点と後から加筆した時点で整合を取っていなかったことを示している。今後の記事作成でも「後から書き足した部分が冒頭の要約と矛盾していないか」を意識すべき教訓。

## 理由

いずれも「間違った情報のまま本番に向かう」という、当初からのファクトチェック計画の目的に直結する実害のある誤り。特に精算表と伝票の誤りは、受験生が自分で検算した際に矛盾に気づき、学習の混乱・信頼低下につながる。

## 影響

- 精算表・伝票・補助簿・仕訳日計表・銀行勘定調整表の5記事の正確性が向上し、受験生が自己検算で混乱するリスクが解消される
- 伝票記事から範囲外の内容が除かれ、無駄な学習負担が減る
