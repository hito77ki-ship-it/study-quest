# 意思決定ログ（ADR）

意思決定は以下の形式で記録する。規模の大きい提案（デザイン変更・新機能・記事コンテンツ戦略など）はこのフローを通す。軽微な変更（誤字修正など）は対象外。

## ファイル命名規則

`YYYYMMDD-[タイトル].md`

例：`20260409-会社設立.md`

## テンプレート

```markdown
# [タイトル]

**日付：** YYYY年MM月DD日
**決定者：** CEO ジョブズ / 取締役承認
**ステータス：** 検討中 / 採用 / 却下

## 背景
なぜこの決定が必要になったか

## リサーチ要点
ファクトチェックリサーチで集めた関連情報・データ・ベストプラクティスの要点

## 決定内容
何を決めたか

## ジョブズの所見
「それはユーザーの人生を本当に変えるか？」の観点での判断。シンプルさ優先、データは参照するが従属しない

## 理由
なぜこの選択をしたか。他の選択肢とのトレードオフ

## 影響
この決定がプロダクト・ユーザー・会社に与える影響

## フォローアップ
次のアクション
```
