# 銀行勘定調整表の級誤表記の修正と重複記事の統合

**日付：** 2026年7月19日
**提案者：** 取締役
**決定者：** CEO ジョブズ / 取締役承認済み
**ステータス：** 承認・実行済み

## 発端

取締役からの一言の疑問がきっかけ：「銀行勘定調整表って、日商簿記3級範囲内なん？」

## ファクトチェック

WebSearchで確認したところ、銀行勘定調整表は日商簿記検定の出題範囲としては**2級**の内容であり、`boki3-ginko.html`（および転載元のnote⑨）が「簿記3級」として掲載していたのは事実誤認だった。

さらに調査を進めると、より深刻な問題が見つかった。サイト内には既に `boki2-ginko-chosei.html`（簿記2級・同一トピック「銀行勘定調整表」を4パターン仕訳の要否まで踏み込んで解説、407行）が存在しており、`boki3-ginko.html`（365行）と内容がほぼ重複していた。単に級表記を「3級」→「2級」に直して`index.html`のカテゴリカードを移すだけでは、同一キーワードを狙う記事がホームページの同じカテゴリに2本並ぶことになり、company/reports/20260716-AdSenseポリシー実質内容の精読記録.mdで指摘した「重複コンテンツ」ポリシー違反リスクと、SEO上のキーワード共食いを引き起こすと判断した。

## ジョブズの所見

「それはユーザーの人生を本当に変えるか？」という基準で見ると、同じ論点を2本の記事に分けて読ませることにユーザー価値はない。むしろ情報が分散し、どちらが正しい難易度なのか混乱を招くだけ。中途半端に両方残すという妥協はしない。

現在の最優先目標である「検索上位表示」の観点でも、重複コンテンツはGoogleに評価を分散させるだけでマイナス。1本に統合し、リソースを集中させる方が合理的。

## 決定

取締役に4案（301リダイレクトで統合／切り口を差別化して両方残す／boki3-ginko.htmlを削除してindexから外す／一旦indexに載せず保留）を提示し、**「boki3-ginko.htmlを301リダイレクトして統合する」**を選択・承認。

### 実行内容

1. `boki3-ginko.html`の本文を全て「銀行勘定調整表の解説はこちらに移動しました」という統合済み案内ページに置き換え、`https://study-quest.net/boki2-ginko-chosei.html`へ`meta refresh`（0秒）+ `canonical`タグでリダイレクト。GitHub Pagesは静的ホスティングのためサーバーサイド301は組めず、この形が実質的な代替手段。`noindex`も付与し、検索エンジンに旧URLではなく統合先を評価させる。
2. `index.html`の`cat-boki3`セクションから`boki3-ginko.html`へのカードを削除（`cat-boki2`側には元々`boki2-ginko-chosei.html`のカードが存在するため、新規追加は不要）。
3. `boki3-progress.html`（簿記3級の学習進捗チェックリスト）から`boki3-ginko.html`の行を削除。`boki2-progress.html`側には元々`boki2-ginko-chosei.html`が正しく登録済みであることを確認。

### 取締役への申し送り事項（要対応）

- **note⑨**（`boki3-ginko.html`を要約・転載したnote記事）は取締役ご自身でnote.com上のリンク・本文の級表記を手動修正する必要がある。統合先の`boki2-ginko-chosei.html`へのリンクに差し替えることを推奨。
- 修正をサイトに反映するには、いつも通りご自身のMacターミナルで以下を実行してデプロイする：
  ```
  cd ~/study-quest
  git add -A
  git commit -m "銀行勘定調整表の級誤表記を修正し、重複記事をboki2-ginko-choseiに統合"
  git push
  ```

## 影響範囲

- 変更ファイル：`boki3-ginko.html`（全面リダイレクト化）、`index.html`（カード削除）、`boki3-progress.html`（行削除）
- URLは維持（`boki3-ginko.html`は削除せず、リダイレクトページとして残置）→ インデックス回復期間中の新規URL発行リスクを回避
- 外部からの被リンク・ブックマークがあってもリダイレクトで着地先が保証される
