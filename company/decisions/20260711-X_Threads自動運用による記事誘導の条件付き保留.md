# X/Threads自動運用による記事誘導・広告展開の検討と条件付き保留

**日付：** 2026年7月11日
**決定者：** CEO ジョブズ / 取締役
**ステータス：** 保留（条件付き・トリガー設定済み）

## 背景

取締役がInstagramのリールで「Claude Code × X自動運用」「無職でも月300万」等のAI×SNS収益化を謳う投稿を複数見かけ、Study Questでも真似できないか、また「就活をしたくないのでこれで生計を立てられないか」という相談があった。

## リサーチ・検証内容

見せられた投稿群（Instagram Photos Libraryから16枚＋34枚、計2バッチ）を実際に確認したところ：

- 「無職でも毎月300万」系の投稿は、ぼかした銀行残高画面のみを証拠とし、**収益化の核心（どうやってその投稿がお金に変わるのか）を一切説明していない**。手法自体（Claudeに悩み×行動チェックリスト形式の投稿文を生成させる）は再現可能だが、収益の再現性は検証不能。
- 「claude code × X自動運用」系は、ターミナル画面を模した演出で「followers +12,480/7d」等の数字を見せているが、表示されているコード（`if score > 0.78: self.x.post(draft)`）は実際に機能するようなものではなく、演出目的のダミーコードと判断。
- 一部（DOTAI）は`広告`と明記されたリード獲得広告で、「この動画が表示された方にだけ」等の定番の煽り文句を使用しており、個人情報登録から有料コース営業への誘導が疑われる。

技術的な実現可能性については切り分けて整理：
- 投稿文の生成自体は現在のClaude Codeサブスクリプションの範囲内で無料で可能
- 完全自動投稿（無人でのポスト）にはX/Threads側の有料APIまたは開発者登録が別途必要

## ジョブズの所見

「それはユーザーの人生を本当に変えるか？」の基準で見ると、今すぐ着手すべきではないと判断した。理由は3つ。

1. 検証不能な収益証拠を根拠に、既に動き出している現実的な選択肢（みそら税理士法人のインターン応募）を天秤にかけて後者を選ぶのは、リスクの取り方として筋が悪い。[[feedback_user_study_vs_content_balance]]と同じ「楽しい方に流れる」パターンが、より大きな賭け（就活そのもの）に対して起きかけていた。
2. 自動化には実コストが発生する（X API有料プラン等）。Study Questの記事制作・アプリ改善にリソースを集中すべき今、検証前の段階でコストを積むのは順序が逆。
3 . 何より本人が「資格勉強の時間が食われそう」と自覚し、取締役自身の判断で保留を選んだ。この自己判断は正しく、CEOとして追認する。

ただし、アイデア自体（Threadsを「わからない」キーワード戦略の記事への導線として使う）は筋が悪いわけではない。取締役から「将来ある程度収益が発生し、伸びる見込みが立った段階で再提案してほしい」という要望があったため、条件付きの保留としてトリガーを設定する。

## 影響

- 今回は着手せず。技術検証・自動化への投資は行わない
- Study Questの記事執筆・資格勉強のバランスを優先する
