# 内部リンク孤立記事（オーファンページ）の解消

**日付：** 2026年7月13日
**提案者：** 取締役（「トップページに全記事入ってる？」という質問がきっかけ）
**決定者：** CEO ジョブズ / 取締役
**ステータス：** 承認・実行中（2026-07-13）

## 背景

取締役から「トップページに全記事が入っているか、アドセンス合格には全記事が必要か」と質問があった。調査した結果、index.htmlの記事一覧セクションに直接リンクされているのは198本中148本で、51本が直接リンクなしと判明。ただし多くは`toeic.html`→`toeic-XXX.html`のようにハブページ経由で2ホップで到達可能だったため、「index.htmlに直接リンクがあるか」ではなく「**index.htmlから何ホップかけても本当に到達不能か**」を機械的に検証した。

## 調査方法・結果

全htmlファイル間の`href`リンクを抽出し、`index.html`を起点にBFS（幅優先探索）でグラフ到達可能性を検証するPythonスクリプトを実行。

**結果：全205ファイル中、index.htmlから完全に到達不能な記事が19本**（ゲーム・チラシ等の非記事ページを除く）判明。

| カテゴリ | 記事 | 原因 |
|---|---|---|
| TOEIC | toeic-600/730/860/900.html（4本） | `toeic.html`本文にスコア別深掘り記事へのリンクが存在しない |
| 英検 | eiken2/3/4/5.html、eiken-jun2.html、eiken3-grammar.html（6本） | `eiken.html`（英検2級記事）に級別・文法記事へのリンクが存在しない |
| IELTS | ielts.html（1本） | どこからもリンクなし |
| 簿記2級 | boki2-cbt-vs-paper/boki2-kotei-shisan-why/boki2-next-exam.html（3本） | index.htmlのcat-boki2カードに含まれず |
| 簿記3級 | boki3-cbt-vs-paper/boki3-next-exam.html（2本） | 同上 |
| 大学生系 | univ-ai-report-hack/univ-boki-units/univ-qualification-strategy.html（3本） | 既存の大学生系導線（daigakusei-keizoku.html等）から漏れている |

これらはsitemap.xmlには含まれているためGoogleに発見される可能性はゼロではないが、内部リンクによる発見経路がなく、今日実施したインデックス状況チェックでも同様に大量の未インデックスが確認されている（[[project_seo_indexing_baseline]]）ことと整合する。

## ジョブズの所見

「それはユーザーの人生を本当に変えるか？」以前の話として、これは**単純にサイトの構造的なバグ**。記事を書いても、サイト内のどこからも辿り着けなければ書いていないのと同じ。Googleのクロールも人間の回遊も、リンクを辿って発見するのが基本経路であり、sitemap.xmlだけに頼るのは補助輪でしかない。

アドセンス審査基準に「全記事をトップページに」という直接の項目はないが、「ナビゲーションが機能しているか」は見られる観点であり、19本が事実上どこからも辿れない状態は望ましくない。

**結論：承認。** 19本それぞれを最も自然な導線（既存ハブページの本文中、またはindex.htmlの該当カテゴリカード）に追加する。新しいセクションを大量新設するのではなく、既存の構造を活かした最小限の追加で直す。

## 実施方針

1. `toeic.html`：本文中の「スコア別攻略法」セクション付近にスコア別4記事へのリンクカードを追加
2. `eiken.html`：本文中に級別・文法記事6本へのリンクカードを追加
3. `index.html` cat-lang：`ielts.html`のカードを追加
4. `index.html` cat-boki2：3本のカードを追加
5. `index.html` cat-boki3：2本のカードを追加
6. `index.html` cat-more：大学生系3本のカードを追加
7. 修正後、同じBFSスクリプトで再検証し、孤立0件を確認
8. スクリーンショットで表示崩れがないことを確認

## 影響

- 対象：`index.html`、`toeic.html`、`eiken.html`への内部リンク追加のみ。新規記事コンテンツの作成・改変は無し
- リスク：低（リンク追加のみ、既存デザイン・構造は維持）

## 副次的な発見（別途対応が必要）

作業中、`eiken.html`と`eiken2.html`が**タイトル完全一致**（「英検2級の勉強法・合格スケジュール【2026年版】」）かつ両方が別々にcanonical自己参照していることが判明。中身は別記事（構成が異なる）だが、Googleから見ると同一キーワードを狙う重複タイトルページに見え、note.comの件と同種のカニバリゼーションリスクがある。これは今回のスコープ外（リンク追加のみ）なので、別途稟議にかけて判断する。
