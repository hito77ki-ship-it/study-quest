# Tier2 簿記2級商業簿記バッチ（9本）ファクトチェック結果

**日付：** 2026年7月8日
**決定者：** CEO ジョブズ / 取締役承認
**ステータス：** 採用

## 背景

Tier2の2級商業簿記バッチ（決算整理・財務諸表／商品売買・棚卸資産／純資産・剰余金配当／法人税等・消費税／ソフトウェア／銀行勘定調整表／有価証券／引当金／社債）9本を並行フォークでファクトチェックした。

## リサーチ要点

| 記事 | 結果 |
|------|------|
| boki2-kessan.html（決算整理・財務諸表） | 構造的な過不足：図解キャプションで「決算整理6大事項」と謳うが、本文で説明されているのは4項目のみ。有価証券・法人税等は名前だけで内容がなく、既存の専用記事（boki2-yukashoken.html、boki2-hojinzei-shohizei.html）への内部リンクも欠けている |
| boki2-shohin-baibai.html（商品売買・棚卸資産） | 実質的な抜け：棚卸減耗損・商品評価損の損益計算書上の表示区分（原価性の有無による判定基準）が一切説明されていない |
| boki2-junshisan.html（純資産・剰余金配当） | 計算・利益準備金ルールは正確（3級で見つかった誤りはこちらには無し）。壊れた内部リンク（存在しない`boki2-kabushiki.html`）を発見 |
| boki2-hojinzei-shohizei.html（法人税等・消費税） | 問題なし |
| boki2-software.html（ソフトウェア） | **重大な出題範囲の誤り**：「市場販売目的ソフトウェア」は実際は日商簿記1級の範囲。この記事ではtitle・meta description・「3分類（★最重要）」・trap-box・FAQなど記事の約半分がこの範囲外の内容で構成されており、2級で本来学ぶべき「自社利用ソフトウェア」が埋もれている |
| boki2-ginko-chosei.html（銀行勘定調整表） | 誤り（計算）：計算例で会社帳簿残高480,000＋未渡小切手40,000－銀行手数料10,000＝510,000となるはずが、表示は500,000（銀行側の調整後残高と帳尻を合わせた数字）。3級で見つかった未渡小切手の分類ミスはこちらには無し |
| boki2-yukashoken.html（有価証券） | 問題なし |
| boki2-hikiatekin.html（引当金） | 実質的な抜け：2級の正式な出題範囲である「一般債権（貸倒実績率法）／貸倒懸念債権・破産更生債権等（財務内容評価法）」の区分が説明されていない。退職給付引当金も簡便法の計算構造・年金掛金拠出の仕訳が扱われていない |
| boki2-shasai.html（社債） | 問題なし |

Sources: [自社利用のソフトウェアの仕訳と会計処理（暗記不要の簿記独学講座）](https://wwboki.jp/dokuboki/commercial-2kyu/software/)／[棚卸減耗損と商品評価損（いぬぼき）](https://inuboki.com/2q-syoubo-kouza/kobetu2-2/)／[債権の区分（一括評価と個別評価）（いぬぼき）](https://inuboki.com/2q-syoubo-kouza/kobetu7-1/)

## 決定内容

1. `boki2-software.html`：「市場販売目的ソフトウェア」を1級範囲の参考情報として明確に格下げし、「自社利用ソフトウェア」を2級の本題として再構成する
2. `boki2-ginko-chosei.html`：計算例の会社帳簿残高（480,000→470,000）を修正し、銀行側と一致させる
3. `boki2-hikiatekin.html`：一般債権・貸倒懸念債権等の区分と財務内容評価法、退職給付引当金の簡便法・年金掛金拠出の仕訳を追加
4. `boki2-shohin-baibai.html`：棚卸減耗損・商品評価損の表示区分（原価性の有無による判定基準）を追加
5. `boki2-kessan.html`：「6大事項」の看板と本文の説明範囲を一致させ、有価証券・法人税等の専用記事への内部リンクを追加
6. `boki2-junshisan.html`：壊れたリンク（`boki2-kabushiki.html`）を修正

## ジョブズの所見

ソフトウェアの記事は今回で一番重い発見。誤情報というより「不要な情報が主役になっている」パターンで、2級受験生が試験に出ない1級範囲を「最重要論点」だと思い込んで時間を溶かすリスクがある。これは合格率に直結する実害。

引当金・商品売買・決算整理の3本に共通するのは、「間違ってはいないが、2級として問われる深さまで届いていない」という抜けのパターン。3級Tier2で見つかった誤りとは性質が違う——3級は主に「計算ミス」、2級では「範囲の判断ミス・深さ不足」が目立つ。難易度が上がるほど、抜けが見つかりやすいAI生成の弱点が出ているのかもしれない。次回以降の2級バッチでも同じ傾向を警戒する。

## 影響

- ソフトウェア記事の修正により、2級受験生が範囲外の内容に時間を使うリスクが解消される
- 引当金・商品売買の抜けが埋まり、2級本試験での対応力が上がる
